OSI
1계층 물리계층
 - 물리적 연결, 전기 신호 전달에 초점
 - 주요 장비 : 허브, 리피터, 케이블, 커넥터, 트랜시버, 탭 등
 - 주소의 개념이 없어 전기 신호가 포트를 제외하고 모든 포트에 같은 전기 신호를 전송

2계층 데이터 링크 계층
 - 전기 신호를 데이터 형태로 처리
 - 주소 정보 정의, 출발지와 도착지 확인 및 데이터 처리 수행
 - 여러 통신이 이루어짐, 여러 명과 통신 할 수 있음
 - 일반적으로 에러 탐지 및 고침, 신뢰 가능할 경우 재전송 까지
 - 플로 컨트롤 : 받는 사람이 데이터 받을 수 있는 상태인지 확인
 - 네트워크 구성요소 : 네트워크 인터페이스 카드, 스위치
 - MAC 주소 체계, 이를 보고 통신 포트 지정해서 신호를 보냄

 - 네트워크 인터페이스 카드란
   (네트워크 인터페이스 카드 = 네트워크 인터페이스 컨트롤러 = NIC = 네트워크 카드 = 랜 카드 = 물리 네트워크 인터페이스 = 이더넷 카드 = 네트워크 어댑터)
   고유 MAC 주소 있음, 자신에게 맞는 신호인지 확인 및 메모리 적재 아니면 버림,

 - 전체 네트워크에서 동시에 오직 하나의 장비만 데이터를 보낼 수 있음
 - 스위치의 역할로 필요한 포트만 사용, 불필요한 처리 감소, 이더넷 네트워크 효율성 향상

3계층 네트워크 계층
 - 장비 : 라우터
 - 라우터 : IP 주소를 사용해서 경로를 찾아 패킷으 전송함
 - IP 구조 :네트워크 부분 호스트 부분
 - 프로토콜 : ARP, IPv4, 6, NAT, IPSec, 라우팅 프로토콜

4계층 트랜스포트 계층(전송)
 - 하위 계층의 데이터를 정확한 곳에 전송을 중점으로 한다.
 - 순서 조절, 재전송, ACK 번호, 시퀸스 번호
 - 장치 내의 많은 어플리케이션 구분 할 수 있도록 포트 번호를 사용해서 구분한다.
 - 장비 : 로드 밸런서, 방화벽
 - 프로토콜 : TCP, UDP 등

5계층 세션 계층
 - 양 끝단 프로세스 연결 및 유지, 종료
 - 에러 복구 및 재전송

6계층 프레젠테이션 계층 (표현 계층)
 - 어플리케이션과 시스템 간의 통신을 돕기 위함
 - 변환기 역할
 - MIME 인토딩, 암호화, 압축, 코드 변환기

7계층 애플리케이션 계층
 - 프로세스 정의 및 어플리케이션 서비스 수행
 - 사용자 UI 정의
 - 대표적 프로토콜 : FTP, SMTP, HTTP, TELNET

인캡슐레이션
 - 데이터를 보내는 과정
디캡슐레이션
 - 데이터를 받는 과정

인캡슐레이션과 디캡슐레이션
 - 위 과정을 통해 데이터가 전송
 - 송수신자 간의 논리적 통신 과정

 - 출발지 : 상위 계층에서 패킷 형태로 하나씩 인캡슐레이션되면서 내려오고 랜 카드에서 전기 형태로 변환 되어 목적지로 전달
 - 도착지 : 전기 신호 받은 것을 데이터 형태로 변환해 상위 계층으로 올려주고 이 패킷을 조합해 데이터 형태로 만듬

 - 헤더 추가 되면 해당 추가 계층에 해당하는 계층에서 헤더를 확인한다.

 - 2계층 헤더 : 도착지 MAC, 소스 MAC, 이더 타입
 - 3계층 헤더 : 버전, 길이, Flags, Fragment, protocol, checksum, TTL, IP 주소, 목적지 IP 주소... 등
 - 4계층 헤더 : 포트, 도착지 포트, 시퀸스 넘버, ACK 넘버, Window Size, checksum, flags, data offset... 등

 - 헤더에 두 가지 정보는 반드시 포함 (현재 계층에서 정의하는 정보, 상위 프로토콜 지시자)

 - 4계층 : 데이터 조립 및 분할		/ 포트 번호
 - 3계층 : IP 주소를 헤더에 적음 출발, 도착	/ 프로토콜 번호
 - 2계층 : MAC 주소를 헤더에 적음 출발, 도착	/ 이더 타입

 - MSS & MTU (둘다 데이터 크기 조절)
 - MSS : maximum segment size
 - MTU : maxumum transmission uint
